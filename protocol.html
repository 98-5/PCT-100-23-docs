📬 Topic 定义

上行：状态上报（设备 → 服务器）

Topic:  chemctrl/{device_id}/status
方向:  设备 → 服务器
触发:  状态变化事件（温度 > 0.5℃ / 光照 > 20 lux / 按键 / 继电器 / 模式改变）
QoS:   0

Payload 示例

{
  "temperature": 28.5,
  "light": 476,
  "mode": "auto",
  "key1_lock": true,
  "relay3": false,
  "relay4": true,
  "temp_threshold": 31.0,
  "light_threshold": 200
}
字段类型说明
temperaturefloat当前温度 (℃)，精度 0.1
lightint当前光照 (lux)
modestring"auto" 自动 / "manual" 手动
key1_lockboolKEY1 总闸：true=开启 false=锁定
relay3bool灯光继电器：true=ON false=OFF
relay4bool风扇继电器：true=ON false=OFF
temp_thresholdfloat风扇开启温度阈值 (℃)
light_thresholdint灯光开启光照阈值 (lux)

下行：控制指令（服务器 → 设备）

Topic:  chemctrl/{device_id}/command
方向:  服务器 → 设备（或 PC 上位机 → 服务器 → 设备）
QoS:   0

🎮 远程控制指令

1. 控制继电器 — set_relay

{
  "cmd": "set_relay",
  "relay": 3,
  "value": true
}
字段说明
relay3 = 灯光，4 = 风扇
valuetrue = 开启，false = 关闭
⚠️ KEY1 关闭时 set_relay 指令被忽略。手动模式下才可控制继电器。

2. 切换模式 — set_mode

{"cmd": "set_mode", "mode": "auto"}
{"cmd": "set_mode", "mode": "manual"}
mode说明
"auto"切换到自动模式（传感器自动控制继电器）
"manual"切换到手动模式（按键或远程控制继电器）
⚠️ KEY1 关闭时 set_mode 指令被忽略，ESP32 每轮 loop() 都会强制切回 auto。

3. 查询状态 — get_status

{"cmd": "get_status"}

设备收到后立即上报一次完整状态到 chemctrl/{id}/status。

4. 设置阈值 — set_threshold

{
  "cmd": "set_threshold",
  "temp": 32.0,
  "light": 250
}
字段说明
temp风扇开启温度阈值 (℃)，关闭阈值自动 = temp - 1℃
light灯光开启光照阈值 (lux)，关闭阈值自动 = 67% 开启阈值
兼容字段别名：temperature / temp_threshold 等同于 temp；lux / light_threshold 等同于 light。

修改后自动保存到 Flash，返回新的 status 确认。

5. 远程重启 — reboot

{"cmd": "reboot"}

设备延时 100ms 后调用 ESP.restart()。

📋 指令速查表

cmd关键参数说明
set_relayrelay: 3/4, value: true/false控制继电器（受 KEY1 和模式约束）
set_modemode: "auto"/"manual"切换工作模式（受 KEY1 约束）
get_status—查询设备状态
set_thresholdtemp: 数值, light: 数值设置温度/光照阈值
reboot—远程重启设备

⚡ 状态上报触发条件

触发条件阈值
温度变化> 0.5 ℃
光照变化> 20 lux
KEY1 状态改变任意变化
灯光继电器改变任意变化
风扇继电器改变任意变化
模式切换任意变化
收到 get_status立即上报
收到 set_threshold保存后上报
set_relay 成功立即上报
